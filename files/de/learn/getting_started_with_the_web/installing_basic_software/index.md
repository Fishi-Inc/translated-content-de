In _Installation von grundlegender Software_ zeigen wir Ihnen, welche Werkzeuge Sie für einfache Webentwicklung benötigen und wie Sie diese richtig installieren.

## Welche Werkzeuge benötige ich gerade wirklich?

In diesem Artikel richten wir Sie mit dem Nötigsten ein — einem Texteditor und einigen modernen Webbrowsern.

### Installation eines Texteditors

Sie haben wahrscheinlich bereits einen einfachen Texteditor auf Ihrem Computer. Standardmäßig enthält Windows [Notepad](https://en.wikipedia.org/wiki/Microsoft_Notepad) und macOS kommt mit [TextEdit](https://en.wikipedia.org/wiki/TextEdit). Linux-Distributionen variieren; die Ubuntu 22.04 LTS-Version enthält standardmäßig den [GNOME Text Editor](https://en.wikipedia.org/wiki/GNOME_Text_Editor).

Für die Webentwicklung können Sie wahrscheinlich besser als Notepad oder TextEdit abschneiden. Wir empfehlen, mit [Visual Studio Code](https://code.visualstudio.com/) zu beginnen, einem kostenlosen Editor, der Live-Vorschauen und Code-Hinweise bietet.

### Installation moderner Webbrowser

Für den Anfang installieren wir ein paar Desktop-Webbrowser, um unseren Code zu testen. Stellen Sie sicher, dass Sie mindestens zwei dieser Browser aus zwei verschiedenen Familien installieren (damit Sie nicht in zwei Browsern testen, die denselben Rendering-Engine verwenden):

- Chromium: [Chrome](https://www.google.com/chrome/), [Opera](https://www.opera.com/), [Brave](https://brave.com/), [Microsoft Edge](https://www.microsoft.com/en-us/edge)
- Gecko: [Firefox](https://www.mozilla.org/en-US/firefox/new/)
- WebKit: [Safari](https://www.apple.com/safari/) (nur macOS und iOS)

### Installation eines lokalen Webservers

Einige Beispiele müssen von einem Webserver ausgeführt werden, um erfolgreich zu funktionieren. Wie das geht, erfahren Sie in [Wie richten Sie einen lokalen Testserver ein?](/de/docs/Learn/Common_questions/Tools_and_setup/set_up_a_local_testing_server)

## Welche Werkzeuge verwenden die Profis?

Die folgende Liste sieht vielleicht beängstigend aus, aber glücklicherweise können Sie ohne Kenntnisse der meisten dieser Werkzeuge mit der Webentwicklung beginnen.

- **Ein Computer**. Das mag für einige offensichtlich klingen, aber einige von Ihnen lesen diesen Artikel auf Ihrem Telefon oder einem Computer in der Bibliothek. Für ernsthafte Webentwicklung ist es besser, in einen Desktop- oder Laptop-Computer zu investieren, der Windows, macOS oder Linux ausführt.
- **Ein Texteditor**, um Code zu schreiben. Dies könnte ein Texteditor sein (z.B. [Visual Studio Code](https://code.visualstudio.com/), [Notepad++](https://notepad-plus-plus.org/), [Sublime Text](https://www.sublimetext.com/), [GNU Emacs](https://www.gnu.org/software/emacs/) oder [VIM](https://www.vim.org/)), oder ein hybrider Editor (z.B. [Dreamweaver](https://www.adobe.com/products/dreamweaver.html) oder [WebStorm](https://www.jetbrains.com/webstorm/)). Office-Dokument-Editoren sind hierfür nicht geeignet, da sie auf versteckte Elemente angewiesen sind, die mit den von Webbrowsern verwendeten Rendering-Engines interferieren.
- **Webbrowser**, um den Code zu testen. Derzeit sind die meistgenutzten Browser [Firefox](https://www.mozilla.org/en-US/firefox/new/), [Chrome](https://www.google.com/chrome/), [Safari](https://www.apple.com/safari/) und [Microsoft Edge](https://www.microsoft.com/en-us/edge). Sie sollten auch testen, wie Ihre Website auf mobilen Geräten und in anderen Browsern, die Ihre Zielgruppe möglicherweise verwendet, funktioniert. [Lynx](https://lynx.browser.org/), ein textbasierter Terminal-Webbrowser, ist hervorragend, um zu sehen, wie Ihre Website von sehbehinderten Nutzern erlebt wird.
- **Ein Grafikeditor**, wie [GIMP](https://www.gimp.org/), [Figma](https://www.figma.com/), [Paint.NET](https://www.getpaint.net/), [Photoshop](https://www.adobe.com/products/photoshop.html), [Sketch](https://www.sketch.com/) oder [XD](https://helpx.adobe.com/support/xd.html), um Bilder oder Grafiken für Ihre Webseiten zu erstellen.
- **Ein Versionskontrollsystem**, um Dateien auf Servern zu verwalten, in einem Team an einem Projekt zusammenzuarbeiten, Code und Ressourcen zu teilen und Bearbeitungskonflikte zu vermeiden. Derzeit ist [Git](https://git-scm.com/) das beliebteste Versionskontrollsystem zusammen mit dem Hosting-Service [GitHub](https://github.com/) oder [GitLab](https://about.gitlab.com/).

Für weitere Informationen über andere projektspezifische Werkzeuge, insbesondere für die clientseitige Webentwicklung, lesen Sie unseren [Verstehen der clientseitigen Webentwicklungstools](/de/docs/Learn/Tools_and_testing/Understanding_client-side_tools) Leitfaden.
