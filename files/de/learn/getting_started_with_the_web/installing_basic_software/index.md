In _Installation von Basissoftware_ zeigen wir Ihnen, welche Werkzeuge Sie für einfache Webentwicklung benötigen und wie Sie diese ordnungsgemäß installieren.

## Welche Werkzeuge brauche ich tatsächlich, jetzt sofort?

In diesem Artikel stellen wir Ihnen das Nötigste bereit — einen Texteditor und einige moderne Webbrowser.

### Installation eines Texteditors

Sie haben wahrscheinlich bereits einen einfachen Texteditor auf Ihrem Computer. Standardmäßig enthält Windows [Notepad](https://en.wikipedia.org/wiki/Microsoft_Notepad) und macOS kommt mit [TextEdit](https://en.wikipedia.org/wiki/TextEdit). Linux-Distributionen variieren; die Ubuntu 22.04 LTS-Version kommt standardmäßig mit dem [GNOME Text Editor](https://en.wikipedia.org/wiki/GNOME_Text_Editor).

Für Webentwicklungen können Sie wahrscheinlich besser als Notepad oder TextEdit arbeiten. Wir empfehlen, mit [Visual Studio Code](https://code.visualstudio.com/) zu beginnen, einem kostenlosen Editor, der Live-Vorschauen und Code-Hinweise bietet.

### Installation moderner Webbrowser

Für den Moment installieren wir ein paar Desktop-Webbrowser, um unseren Code darin zu testen. Stellen Sie sicher, dass Sie mindestens zwei dieser Browser installieren, aus zwei verschiedenen Familien (damit Sie nicht in zwei Browsern testen, die die gleiche Rendering-Engine verwenden):

- Chromium: [Chrome](https://www.google.com/chrome/), [Opera](https://www.opera.com/), [Brave](https://brave.com/), [Microsoft Edge](https://www.microsoft.com/en-us/edge)
- Gecko: [Firefox](https://www.mozilla.org/en-US/firefox/new/)
- WebKit: [Safari](https://www.apple.com/safari/) (nur macOS und iOS)

### Installation eines lokalen Webservers

Einige Beispiele müssen von einem Webserver ausgeführt werden, um erfolgreich zu funktionieren. Erfahren Sie, wie das geht, in unserer [Anleitung zur Einrichtung eines lokalen Testservers](/de/docs/Learn/Common_questions/Tools_and_setup/set_up_a_local_testing_server).

## Welche Werkzeuge verwenden die Profis?

Die folgende Liste mag abschreckend wirken, aber zum Glück können Sie mit der Webentwicklung beginnen, ohne die meisten davon zu kennen.

- **Ein Computer**. Das mag für manche offensichtlich klingen, aber einige von Ihnen lesen diesen Artikel auf einem Telefon oder einem Bibliothekscomputer. Für ernsthafte Webentwicklung ist es besser, in einen Desktop- oder Laptop-Computer mit Windows, macOS oder Linux zu investieren.
- **Ein Texteditor**, um Code zu schreiben. Dies könnte ein Texteditor (z.B. [Visual Studio Code](https://code.visualstudio.com/), [Notepad++](https://notepad-plus-plus.org/), [Sublime Text](https://www.sublimetext.com/), [GNU Emacs](https://www.gnu.org/software/emacs/) oder [VIM](https://www.vim.org/)) oder ein Hybrid-Editor (z.B. [Dreamweaver](https://www.adobe.com/products/dreamweaver.html) oder [WebStorm](https://www.jetbrains.com/webstorm/)) sein. Office-Dokumenteditoren sind für diesen Gebrauch nicht geeignet, da sie sich auf versteckte Elemente verlassen, die mit den von Webbrowsern verwendeten Rendering-Engines interferieren.
- **Webbrowser**, um den Code zu testen. Derzeit sind die am häufigsten verwendeten Browser [Firefox](https://www.mozilla.org/en-US/firefox/new/), [Chrome](https://www.google.com/chrome/), [Safari](https://www.apple.com/safari/) und [Microsoft Edge](https://www.microsoft.com/en-us/edge). Sie sollten auch testen, wie Ihre Website auf mobilen Geräten und in allen anderen Browsern funktioniert, die Ihre Zielgruppe möglicherweise verwendet. [Lynx](https://lynx.browser.org/), ein textbasierter Terminal-Webbrowser, ist großartig, um zu sehen, wie Ihre Seite von sehbehinderten Nutzern erlebt wird.
- **Ein Grafikeditor** wie [GIMP](https://www.gimp.org/), [Figma](https://www.figma.com/), [Paint.NET](https://www.getpaint.net/), [Photoshop](https://www.adobe.com/products/photoshop.html), [Sketch](https://www.sketch.com/) oder [XD](https://helpx.adobe.com/support/xd.html), um Bilder oder Grafiken für Ihre Webseiten zu erstellen.
- **Ein Versionskontrollsystem**, um Dateien auf Servern zu verwalten, an einem Projekt mit einem Team zusammenzuarbeiten, Code und Ressourcen zu teilen und Bearbeitungskonflikte zu vermeiden. Derzeit ist [Git](https://git-scm.com/) das beliebteste Versionskontrollsystem zusammen mit dem Hosting-Service [GitHub](https://github.com/) oder [GitLab](https://about.gitlab.com/).

Für weitere Informationen über andere projektspezifische Werkzeuge, insbesondere für die clientseitige Webentwicklung, siehe unseren [Leitfaden zu clientseitigen Webentwicklungswerkzeugen](/de/docs/Learn/Tools_and_testing/Understanding_client-side_tools).
