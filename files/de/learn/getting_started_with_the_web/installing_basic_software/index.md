In _Installation grundlegender Software_ zeigen wir Ihnen, welche Werkzeuge Sie für die einfache Webentwicklung benötigen und wie Sie diese ordnungsgemäß installieren.

## Welche Werkzeuge benötige ich tatsächlich gerade jetzt?

In diesem Artikel statten wir Sie mit dem Nötigsten aus: einem Texteditor und einigen modernen Webbrowsern.

### Installation eines Texteditors

Wahrscheinlich haben Sie bereits einen grundlegenden Texteditor auf Ihrem Computer. Windows enthält standardmäßig [Notepad](https://en.wikipedia.org/wiki/Microsoft_Notepad) und macOS beinhaltet [TextEdit](https://en.wikipedia.org/wiki/TextEdit). Linux-Distributionen variieren; die Ubuntu 22.04 LTS-Version kommt standardmäßig mit dem [GNOME Text Editor](https://en.wikipedia.org/wiki/GNOME_Text_Editor).

Für die Webentwicklung können Sie wahrscheinlich besser arbeiten als mit Notepad oder TextEdit. Wir empfehlen den Start mit [Visual Studio Code](https://code.visualstudio.com/), einem kostenlosen Editor, der Live-Previews und Code-Hinweise bietet.

### Installation moderner Webbrowser

Fürs Erste installieren wir ein paar Desktop-Webbrowser, um unseren Code darin zu testen. Stellen Sie sicher, dass Sie mindestens zwei dieser Browser installieren, aus zwei verschiedenen Familien (damit Sie nicht in zwei Browsern testen, die denselben Rendering-Engine verwenden):

- Chromium: [Chrome](https://www.google.com/chrome/), [Opera](https://www.opera.com/), [Brave](https://brave.com/), [Microsoft Edge](https://www.microsoft.com/en-us/edge)
- Gecko: [Firefox](https://www.mozilla.org/en-US/firefox/new/)
- WebKit: [Safari](https://www.apple.com/safari/) (nur macOS und iOS)

### Installation eines lokalen Webservers

Einige Beispiele müssen von einem Webserver ausgeführt werden, um erfolgreich zu funktionieren. Wie Sie dies einrichten können, erfahren Sie in [Wie richten Sie einen lokalen Testserver ein?](/de/docs/Learn/Common_questions/Tools_and_setup/set_up_a_local_testing_server)

## Welche Werkzeuge verwenden die Profis?

Folgendes sieht nach einer erschreckenden Liste aus, aber zum Glück können Sie mit der Webentwicklung beginnen, ohne die meisten dieser Dinge zu kennen.

- **Ein Computer**. Vielleicht klingt das für einige offensichtlich, aber einige von Ihnen lesen diesen Artikel auf Ihrem Telefon oder einem Bibliothekscomputer. Für ernste Webentwicklung ist es besser, in einen Desktop- oder Laptop-Computer zu investieren, der Windows, macOS oder Linux ausführt.
- **Ein Texteditor**, um Code zu schreiben. Dies könnte ein Texteditor sein (z.B. [Visual Studio Code](https://code.visualstudio.com/), [Notepad++](https://notepad-plus-plus.org/), [Sublime Text](https://www.sublimetext.com/), [GNU Emacs](https://www.gnu.org/software/emacs/) oder [VIM](https://www.vim.org/)) oder ein hybrider Editor (z.B. [Dreamweaver](https://www.adobe.com/products/dreamweaver.html) oder [WebStorm](https://www.jetbrains.com/webstorm/)). Office-Dokumenteditoren sind hierfür nicht geeignet, da sie auf versteckte Elemente angewiesen sind, die mit den von Webbrowsern verwendeten Rendering-Engines interferieren.
- **Webbrowser**, um Code darin zu testen. Derzeit sind die meistverwendeten Browser [Firefox](https://www.mozilla.org/en-US/firefox/new/), [Chrome](https://www.google.com/chrome/), [Safari](https://www.apple.com/safari/) und [Microsoft Edge](https://www.microsoft.com/en-us/edge). Sie sollten auch testen, wie Ihre Website auf mobilen Geräten und in allen alten Browsern, die Ihr Zielpublikum möglicherweise noch verwendet (wie z.B. IE 8–10), funktioniert. [Lynx](https://lynx.browser.org/), ein textbasierter Terminal-Webbrowser, ist großartig, um zu sehen, wie Ihre Website von sehbehinderten Nutzern erlebt wird.
- **Ein Grafikeditor**, wie [GIMP](https://www.gimp.org/), [Figma](https://www.figma.com/), [Paint.NET](https://www.getpaint.net/), [Photoshop](https://www.adobe.com/products/photoshop.html), [Sketch](https://www.sketch.com/) oder [XD](https://helpx.adobe.com/support/xd.html), um Bilder oder Grafiken für Ihre Webseiten zu erstellen.
- **Ein Versionskontrollsystem**, um Dateien auf Servern zu verwalten, gemeinsam an einem Projekt zu arbeiten, Code und Assets zu teilen und Bearbeitungskonflikte zu vermeiden. Zurzeit ist [Git](https://git-scm.com/) das beliebteste Versionskontrollsystem zusammen mit dem Hosting-Dienst [GitHub](https://github.com/) oder [GitLab](https://about.gitlab.com/).
- **Ein FTP-Programm**, das auf älteren Webhosting-Konten verwendet wird, um Dateien auf Servern zu verwalten ([Git](https://git-scm.com/) ersetzt zunehmend FTP für diesen Zweck). Es gibt eine Vielzahl von (S)FTP-Programmen wie [Cyberduck](https://cyberduck.io/), [Fetch](https://fetchsoftworks.com/) und [FileZilla](https://filezilla-project.org/).
- **Ein Automatisierungssystem**, wie [Webpack](https://webpack.js.org/), [Grunt](https://gruntjs.com/) oder [Gulp](https://gulpjs.com/), um automatisch sich wiederholende Aufgaben wie das Minifizieren von Code und das Ausführen von Tests durchzuführen.
- Bibliotheken, Frameworks usw., um das Schreiben von allgemeinen Funktionalitäten zu beschleunigen. Eine Bibliothek ist in der Regel eine vorhandene JavaScript- oder CSS-Datei, die Ihnen vage Funktionalität bietet, die Sie in Ihren Code einbinden können. Ein Framework geht in der Regel weiter und bietet ein komplettes System mit einigen benutzerdefinierten Syntaxen, um darauf eine Webanwendung zu schreiben.
- Noch mehr Werkzeuge!
