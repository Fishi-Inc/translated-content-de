**Progressives Enhancement** ist eine Designphilosophie, die darauf abzielt, so vielen Nutzern wie möglich grundlegende Inhalte und Funktionen bereitzustellen, während die bestmögliche Erfahrung nur den Nutzern moderner Browser geboten wird, die in der Lage sind, den gesamten erforderlichen Code auszuführen.

Das Wort _progressiv_ in _progressives Enhancement_ bedeutet, ein Design zu schaffen, das für Nutzer älterer Browser und Geräte mit begrenzten Fähigkeiten eine einfachere, aber dennoch nutzbare Erfahrung bietet, während es gleichzeitig das Benutzererlebnis zu einer ansprechenderen und voll ausgestatteten Erfahrung für Nutzer neuerer Browser und Geräte mit umfangreicheren Fähigkeiten fortschreitet.

[Feature-Erkennung](/de/docs/Learn/Tools_and_testing/Cross_browser_testing/Feature_detection) wird in der Regel verwendet, um festzustellen, ob Browser mit moderneren Funktionen umgehen können, während {{Glossary("polyfill", "Polyfills")}} häufig verwendet werden, um fehlende Funktionen mit JavaScript hinzuzufügen.

Besondere Aufmerksamkeit sollte der Barrierefreiheit gewidmet werden. Wenn möglich, sollten akzeptable Alternativen bereitgestellt werden.

Progressives Enhancement ist eine nützliche Technik, die Webentwicklern ermöglicht, sich darauf zu konzentrieren, die bestmöglichen Websites zu entwickeln, während sie diese Websites auf mehreren unbekannten Benutzeragenten funktionsfähig machen. {{Glossary("Graceful degradation")}} ist verwandt, aber nicht dasselbe und wird oft als entgegengesetzter Ansatz zum progressiven Enhancement gesehen. Tatsächlich sind beide Ansätze gültig und können sich oft gegenseitig ergänzen.
