**Progressive Enhancement** ist eine Design-Philosophie, die eine Basislinie von essentiellen Inhalten und Funktionen für so viele Benutzer wie möglich bereitstellt, während sie nur den Benutzern der modernsten Browser, die den gesamten erforderlichen Code ausführen können, das bestmögliche Erlebnis bietet.

Das Wort _progressive_ in _Progressive Enhancement_ bedeutet, ein Design zu schaffen, das eine einfachere, aber dennoch nutzbare Erfahrung für Benutzer älterer Browser und Geräte mit eingeschränkten Möglichkeiten bietet, während es gleichzeitig ein Design ist, das **die Benutzererfahrung zu einem ansprechenderen, voll ausgestatteten Erlebnis für Benutzer neuerer Browser und Geräte mit reicheren Möglichkeiten fortschreitet**.

[Feature Detection](/de/docs/Learn/Tools_and_testing/Cross_browser_testing/Feature_detection) wird in der Regel verwendet, um festzustellen, ob Browser modernere Funktionen ausführen können, während [Polyfills](/de/docs/Glossary/polyfill) häufig verwendet werden, um fehlende Features mit JavaScript hinzuzufügen.

Besondere Aufmerksamkeit sollte der Barrierefreiheit gewidmet werden. Akzeptable Alternativen sollten, wo möglich, bereitgestellt werden.

Progressive Enhancement ist eine nützliche Technik, die es Webentwicklern ermöglicht, sich darauf zu konzentrieren, die bestmöglichen Websites zu entwickeln, während sie diese Websites auf mehreren unbekannten User Agents funktionsfähig machen. [Graceful Degradation](/de/docs/Glossary/Graceful_degradation) ist damit verwandt, aber nicht dasselbe und wird oft als in die entgegengesetzte Richtung zum Progressive Enhancement gehend angesehen. In Wirklichkeit sind beide Ansätze gültig und können sich oft gegenseitig ergänzen.
